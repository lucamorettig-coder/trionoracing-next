Note di design · scope MVP EVO-020

  1. Scope filtri: Anno (pills, default anno corrente) + Mese (dropdown, default mese corrente) + Maestro (single select da getAllMaestriAttivi()) + Search bambino (debounced 300ms). Counter risultati a destra come pattern EVO-016/017/018/019.
  2. ❌ NO filtro Corso: confermato da reference Cowork persistente — Triono ha un solo corso MTB/BDC combinato. Non aggiungere il filtro nemmeno disabled.
  3. KPI top: 3 metriche aggregate sul periodo filtrato. "Maestro più attivo" in valueTone="success" grass-700 per dare risalto al riconoscimento.
  4. Click riga → apre LezioneDetailModal read-only (no pagina dedicata, niente modifica/elimina in MVP — le lezioni sono di proprietà del maestro che le ha registrate dal portale).
  5. Maestri chips: lista compatta con avatar iniziali colorate (max 3 visibili, +N se più). Pattern coerente con cell-person di EVO-019 ma in formato chip pill.
  6. N° bambini: counter grass con icona <Users/>. Lezioni con 0 bambini sono filtrate dal portale lato maestro (errore di compilazione) — non dovrebbero apparire qui.
  7. DataTable (non card grid): coerente con altre liste admin EVO-016/017/018/019. Componente base DataTable<Lezione> riusato.
